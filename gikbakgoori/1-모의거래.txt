모의 거래 기능 구현 계획
1. 기능 개요
각 분석 카드 하단에 모의 거래 기능 추가
매수 신호 강도에 따라 자동으로 모의 거래 실행
거래 내역 테이블로 표시
2. 구현 요소
2.1 UI 구성 요소
모의 거래 토글 버튼: 카드 하단에 배치
모의 거래 설정 패널:
자동 거래 활성화 체크박스
매수 신호 강도 기준값 설정 슬라이더 (0-100%)
거래 내역 테이블:
시간, 가격, 신호강도, 액션 칼럼
최근 5개 거래 표시
2.2 디자인 요구사항
기존 카드 디자인과 일관성 유지
깔끔하고 직관적인 UI
모바일 대응 레이아웃
3. 필요한 코드 수정
3.1 카드 컴포넌트 (card.js)
모의 거래 UI 요소 추가
모의 거래 버튼 클릭 이벤트 핸들러
모의 거래 활성화 토글 이벤트 처리
매수 신호 강도 슬라이더 이벤트 처리
거래 내역 업데이트 함수 추가
3.2 거래 서비스 (trading.js)
모의 거래 상태 관리 (활성화/비활성화)
매수 신호 강도 기준값 저장
모의 거래 실행 함수
거래 내역 저장 및 관리 기능
거래 내역 조회 API
3.3 CSS 스타일 (analysis.css)
모의 거래 컨테이너 스타일
슬라이더 디자인
거래 내역 테이블 스타일
4. 주의사항
기존 코드의 기능을 훼손하지 않도록 주의
코드 변경은 필요한 부분만 최소한으로 수정
브라우저 호환성 고려 (Chrome, Firefox, Safari)
성능 영향 최소화
5. 테스트 계획
모의 거래 UI 표시/숨김 기능 테스트
다양한 매수 신호 강도 값에서의 거래 실행 테스트
거래 내역 테이블 업데이트 확인
여러 카드에서 독립적으로 작동하는지 확인
